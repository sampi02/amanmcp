# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.10.x  | :white_check_mark: |
| < 0.10  | :x:                |

## Reporting a Vulnerability

**Please do NOT open public issues for security vulnerabilities.**

Report security vulnerabilities by emailing: **nirajkvinit@gmail.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Timeline**: Communicated after assessment

### What to Expect

1. We'll confirm receipt of your report
2. We'll investigate and validate the issue
3. We'll work on a fix and coordinate disclosure
4. You'll be credited (if desired) when the fix is released

## Security Best Practices

AmanMCP is designed with security in mind:

- **Local-first**: All data stays on your machine
- **No telemetry**: No data sent to external services
- **No network calls**: Search and indexing are 100% local
- **Minimal dependencies**: Reduced attack surface

## Scope

This security policy applies to:
- The AmanMCP CLI tool
- The MCP server component
- Official documentation

Third-party integrations and plugins are outside this scope.
