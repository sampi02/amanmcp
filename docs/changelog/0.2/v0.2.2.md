# v0.2.2 - Unified /aman Command

**Release Date:** 2026-01-10

## Summary

Consolidates 7 PM skills into unified `/aman` command with auto-increment sprint start, sprint-end ceremony, and auto-versioning release.

---

## Added

- Unified `/aman` command consolidating 7 PM skills into 1
  - `/aman start` and `/aman end` for session management
  - `/aman status`, `/aman backlog`, `/aman done` for item tracking
  - `/aman groom`, `/aman report` for sprint planning
  - `/aman arch` for rearchitecture processing
  - `/aman release` for version releases
- Auto-increment for `/aman sprint-start` (reads from config.yaml)
- `/aman sprint-end` command for closing sprints with full ceremony
- Auto-versioning for `/aman release` (detects from VERSION file)
  - `--patch`, `--minor`, `--major` flags for explicit bumps
  - Analyzes unreleased.md to suggest appropriate bump type

## Changed

- Restructure aman-pm backlog directories for consistency
  - Features: flat → active/done/resolved subdirectories
  - Tasks: flat → active/done subdirectories
  - Matches existing bugs/debt structure pattern
- Update generate_index.py to scan new subdirectory structure
- Consolidate session-start, session-end, aman-pm, backlog-groom, sprint-report, rearchitect, release skills into unified /aman

## Fixed

- Flaky TestParser_Performance_Parse1000LOC test (use LessOrEqual for boundary condition)

## Removed

- Deprecated individual PM skills (replaced by unified /aman)

## Documentation

- Update all PM guides to use unified /aman commands
- Add release workflow section to ai-native-pm-guide.md
- Bump guide versions to 2.1.0
